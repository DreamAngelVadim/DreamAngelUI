## 🎯 Обзор

**Wan2.1 Ultimate UI** — это полностью автономное приложение с веб-интерфейсом для генерации видео и изображений с помощью нейросетей семейства Wan2.1. Приложение разработано с фокусом на:

| Ключевая особенность | Описание |
|---------------------|----------|
| **Минимальное потребление ресурсов** | Оптимизировано для видеокарт с 8GB VRAM (RTX 3060 Ti, RTX 3070, RTX 4060 Ti и аналоги) |
| **Полную автоматизацию** | Установка всех зависимостей происходит при первом запуске |
| **Удобство работы** | Интуитивно понятный интерфейс, адаптирующийся под тему Windows |
| **Расширяемость** | Поддержка LoRA, интеграция с CivitAI и Hugging Face |

## 💻 Системные требования

### Минимальные требования

| Компонент | Требование | Примечание |
|-----------|------------|------------|
| **Операционная система** | Windows 10/11 (рекомендуется), Linux, macOS | Полная поддержка всех платформ |
| **Python** | Версия 3.10 или выше | Автоматически проверяется при запуске |
| **Видеокарта** | NVIDIA с 8 GB VRAM | RTX 3060 Ti, RTX 3070, RTX 4060 Ti |

## ✨ Возможности

### 🌟 Основные функции

| Функция | Описание | Время выполнения |
|---------|----------|------------------|
| **Text-to-Video** | Генерация видео из текстового описания (модель 1.3B) | 10-20 минут |
| **Image-to-Video** | Оживление изображения с добавлением движения (модель 14B FP8) | 15-20 минут |
| **Text-to-Image** | Генерация изображений из текстового описания | 2-5 минут |
| **Автоустановка** | Все зависимости устанавливаются автоматически при первом запуске | 5-10 минут |
| **Адаптивная тема** | Интерфейс подстраивается под светлую/тёмную тему Windows | Мгновенно |

### 🎯 Работа с LoRA

| Функция | Описание | Что даёт пользователю |
|---------|----------|----------------------|
| **Поиск LoRA** | Поиск на Hugging Face и GitHub по ключевым словам | Быстрый поиск готовых решений |
| **Авто-обнаружение** | Автоматическое сканирование папки `models/lora` | Не нужно перезапускать приложение |
| **Метаданные** | Чтение триггерных слов, автора, описания из `.safetensors` или `info.json` | Понимание как использовать LoRA |
| **Предпросмотр** | Отображение превью-изображений (локальных или с CivitAI) | Визуальная оценка LoRA до применения |
| **Фильтрация** | Автоматическая фильтрация LoRA по типу задачи (T2V/I2V/T2I) | Показываются только совместимые LoRA |
| **Триггерные слова** | Автоматическая вставка триггерных слов в промпт | Правильная активация LoRA |

### 🤝 Интеграции

| Сервис | Возможности | Ограничения |
|--------|-------------|-------------|
| **Hugging Face** | Скачивание моделей, поиск LoRA | Требуется интернет |
| **CivitAI** | Получение превью, извлечение метаданных (без NSFW фильтра) | Rate limiting (ограничение запросов) |
| **GitHub** | Поиск репозиториев с LoRA | Требуется интернет |

### 🛠️ Системные утилиты

| Функция | Описание | Когда используется |
|---------|----------|---------------------|
| **Автопоиск порта** | Автоматически находит свободный порт, если 7860 занят | При каждом запуске |
| **Автооткрытие браузера** | Браузер открывается автоматически после запуска | Через 2 секунды после старта |
| **Прогресс-бары** | Визуальное отображение прогресса скачивания и генерации | Во всех длительных операциях |
| **Кэширование** | Результаты поиска LoRA кэшируются на 5 минут | При повторных поисках |
| **Докачка файлов** | Прерванные загрузки моделей возобновляются | При обрыве интернета |

⚡ Оптимизация для 8GB VRAM
Приложение использует несколько техник для работы на видеокартах с 8GB VRAM:

1. Offloading (выгрузка модели)


--offload_model True


Как работает: Выгружает неактивные части модели из видеопамяти в оперативную.
Эффект: Экономия ~2 GB VRAM.

2. T5 на CPU

--t5_cpu True


Как работает: Держит текстовый энкодер (T5) в оперативной памяти, а не в видеопамяти.
Эффект: Экономия ~1.5 GB VRAM.

3. FP8 квантование (для I2V)
Используется специальная версия модели Wan2.1-I2V-14B-FP8.
Эффект: Модель занимает ~12 GB VRAM вместо 22 GB.

4. Оптимальные разрешения
Тип задачи	Рекомендуемое разрешение	Максимальное разрешение	Максимальное кадров
T2V 1.3B	832×480	832×832	49
I2V 14B FP8	832×480	832×832	33
T2I	832×832	1024×1024	1
5. Сравнение потребления VRAM
Модель	Без оптимизаций	С offload	С offload + T5 CPU	FP8 версия
T2V 1.3B	~9 GB	~7 GB	~5 GB	~4 GB
I2V 14B	~22 GB	~19 GB	~16 GB	~12 GB

❓ Часто задаваемые вопросы
1. Почему приложение скачивает так много зависимостей?
<details> <summary>Ответ</summary>
Приложение полностью автономно и не требует предварительной установки чего-либо. При первом запуске оно устанавливает все необходимые компоненты автоматически. Это сделано для того, чтобы вы могли просто скачать один файл и запустить его без дополнительных действий.

Что устанавливается:

Python пакеты (gradio, torch, huggingface_hub и др.) — ~2 GB

Репозиторий Wan2.1 — ~50 MB

Зависимости Wan2.1 (diffusers, transformers и др.) — ~500 MB

Итого: ~2.5 GB однократно.

</details>
2. Сколько места занимают модели?
<details> <summary>Ответ</summary>
Модель	Размер	Когда скачивается
T2V 1.3B	~2.8 GB	При первой генерации T2V или T2I
I2V 14B FP8	~16 GB	При первой генерации I2V
Итого	~19 GB	
Модели скачиваются только при первой необходимости. Если вы используете только T2V, I2V не будет скачан.

</details>
3. Почему T2V 14B не работает на 8GB?
<details> <summary>Ответ</summary>
Модель T2V 14B требует ~24 GB видеопамяти даже в режиме инференса (простой генерации). Это связано с архитектурой модели — она содержит 14 миллиардов параметров. Даже с использованием FP8 квантования и offloading, минимальные требования составляют ~12-16 GB VRAM.

Для работы с T2V 14B необходимо:

RTX 4090 (24GB) или выше

Аренда облачного GPU (RunPod, Vast.ai, Lambda Labs)

Использование CPU (очень медленно, часы на одну генерацию)

</details>
